# 🚀 Frontend Project
Проект подразумивает пример онлайн бизнеса. В самом проекте имееьтся регистрация, авторизация, создание записей, база данных и он имеет три разных пользователей:
1. Ownew - Владелец.
2. Admin - Работник.
3. User - Пользователь.
   
   ## 👥 Ролевая модель и возможности

В системе реализовано три уровня доступа:
*   **Owner (Владелец):** 
    *   Полный контроль над платформой.
    *   Управление персоналом (добавление/удаление админов и мастеров).
    *   Просмотр аналитики и финансовой отчетности сервиса.
    *   Настройка глобальных параметров сайта.
*   **Admin (Администратор):** 
    *   Управление текущими записями (подтверждение, перенос, отмена).
    *   Просмотр графиков занятости специалистов.
    *   Модерация клиентской базы.
*   **User (Пользователь):** 
    *   Просмотр списка доступных услуг и специалистов.
    *   Выбор времени и создание записи.
    *   Личный кабинет со своей историей бронирований.
      
## 🗄️ Работа с данными (Database)

В проекте реализована полноценная реляционная база данных. Я самостоятельно спроектировала архитектуру таблиц для обеспечения целостности и быстродействия системы.

*   **Проектирование:** Разработана схема БД с учетом связей «один ко многим» и «многие ко многим» (например, связь пользователей с их ролями и записями).
*   **Реализация:** Бэкенд-часть проекта взаимодействует с БД, обеспечивая безопасное хранение паролей и актуальность статусов бронирования в реальном времени.
*   **Надежность:** Реализована валидация данных на уровне базы, что предотвращает появление некорректных записей или дублей.
  
## ✨ Чем полезен сайт?

*   **Иерархия управления:** Позволяет делегировать задачи администраторам, оставляя стратегический контроль владельцу.
*   **Удобство:** Клиенты записываются самостоятельно, что снижает нагрузку на персонал на 70%.
*   **Масштабируемость:** Структура кода позволяет легко добавлять новые типы услуг и расширять штат сотрудников.
